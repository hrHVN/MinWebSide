Lesson 3 - User Experience


learning outcomes


In this lesson, we are covering the following knowledge learning outcomes:
  • The candidate has knowledge of the concepts, processes, and tools to build static web pages.
In this lesson, we are covering the following skill learning outcome:
  • The candidate masters relevant vocational tools, materials, and techniques to create static web pages using HTML and CSS.
In this lesson, we are covering the following general competence learning outcome:
  • The candidate understands the ethical principles that apply when creating JavaScript web solutions.

To sum up this course, we should:

  1. Keep things simple:
    • all content has a reason for being on the website
    • we used the minimum possible amount of text
    • all of our graphic elements are relevant and show unique information.
  2. Keep things consistent:
    • our menus, headings and all other design elements have the same style within the entire website
    • we presented information in a similar style on the entire website.
  3. Keep things standard:
    • our website looks similar to other websites
    • we used popular solutions for our components, or at least, we made our components behave the same way as the most popular ones
    • our website works correctly on the older browsers as well.
  4. Considered our users:
    • our website was created for users, not us; the user’s primary goal is to find the information they’re looking for as quickly as possible, and we should help them
    • we should think of our target users and present the information in the most convenient way.

The Task

This lesson taught us about standard User Experience (UX) basics. In this task, you need to ask the following questions:

  1. A visitor has reached your website from a search engine. What do you need to consider about the information displayed on the pages?
    • The information on the page should direct the visitor to the site’s homepage, where they can get more information.
    • Each page should outline the site’s content so visitors can access the relevant information.
    • Each page should contain a site map so visitors can find the necessary information.
  2. When simplifying your site’s design, what question should you ask yourself?
    • Is the amount of text on the page sufficient?
    • Should each page contain additional images?
    • Does each element on the page help or hinder visitors?
  3. Before adding different forms of media (video, audio, or interactive content) to your pages, what should you ask?
    • Will this grab people’s attention?
    • Will this slow down page load times?
    • Does this enhance my visitors’ experience?
  4. Adding interactive content to a page can be helpful to demonstrate product features, teach concepts, or give immediate feedback. But what should you keep in mind?
    • You should make sure it’s eye catching so people are encouraged to use it.
    • You should ensure the interactive content is truly educational.
    • You should ensure the interactive content is truly educational.
  5. After developing your information architecture, you realise you have five main menu items. Which type of menu interface are you most likely to choose?
    • A vertical menu on the right-hand side of the page.
    • A horizontal menu just beneath the page header area.
    • A vertical menu on the left-hand side of the page.
