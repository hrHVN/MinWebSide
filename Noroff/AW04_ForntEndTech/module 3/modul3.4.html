Lesson 3 - GIT

In the past chapter, we learned some Git basics, such as standard installation, setup and creating a repository. However, there are many more Git uses, which we’ll learn about in this chapter. First, we’ll learn which conflicts might happen after pulling and pushing changes and how to solve them. Later, we’ll learn more complex Git functionalities, such as creating separate branches, transferring data between them and pulling requests, enabling us to ensure our code is of good quality. We’ll also learn about stashing and how to create a good, readable readme file. After this lesson, you should be able to add source control and use it properly for all your solo and small projects with collaborators.

learning outcomes

In this lesson, we are covering the following knowledge learning outcomes:

  • The candidate has knowledge of concepts, processes, and tools of Git version control software.

    • In this lesson, we are covering the following skill learning outcome:

    • The candidate masters JavaScript tools, materials, techniques, and styles to enhance functionality in static web pages, both inline scripting and external script files.
    • The candidate masters Git source control tools, materials, techniques, and styles to create and work with a version-controlled project.

    In this lesson, we are covering the following general competence learning outcome:

    • The candidate can carry out work based on the needs of a junior JavaScript developer.
    • The candidate understands the ethical principles that apply when creating JavaScript web solutions.

The Task

This lesson taught us about stashing code, creating and merging branches, solving conflicts and creating pull requests. In this lesson task, you need to:

  1. Create a new file new.txt on the main branch, commit and push it.
  2. Create two new branches from main.
  3. In both branches, change the text of the new.txt file.
  4. On the first branch, commit the change, create a pull request and merge the branch into main.
  5. On the second branch, stash this change.
  6. On the first branch, merge changes from main branch.
  7. Apply stash and resolve conflicts, keeping the stashed version.
  8. Commit the changes and push them.
  9. Create a pull request and merge it to the main branch.
  10. On the GitHub platform, delete the two created branches and the new.txt file from the main branch.
