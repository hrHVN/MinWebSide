Lesson 3 - GIT

Git is the state-of-the-art technology for version control in almost any field of programming. Source control lets us see our code history, work on one project simultaneously with other people, or even revert recent changes and restore the old app version without needing a local backup. GitHub is the most popular code hosting platform for version control and collaboration. After creating a GitHub account, we can work together on projects from anywhere. We can also look at other people’s projects. Projects are usually organised in repositories. Repositories are typically private before project completion. After completing the project, we can make them public, show our code to our friends or inspire anyone in the world. In this lesson, we’ll learn how to install Git on our computer and create our GitHub account. We’ll also create our first repository, check the VS Code Git extension, use them to create our first files and push them to the GitHub platform.

The Task

This lesson taught us about Git basics, such as Git setup, creating a GitHub account, repository, and commits. We saw how to push the last to the remote repository.

In this task, you need to:

  1. Create a new repository.
  2. Open it in VS Code.
  3. Create a file “Test.txt” and put some text in it.
  4. Create a commit containing the file creation.
  5. Push the commit to the remote repository.
  6. Change the file again, but from GitHub.
  7. Create another commit with the file change.
  8. Pull remote changes from VS Code.
